# Acionamento do Orquestrador

Use esta tabela para escolher a skill dona da task:

- produto ou escopo -> `product-system-design`
- jornada, telas ou navegacao -> `ui-ux-pro-max`
- app mobile -> `mobile-app-architecture`
- painel admin -> `next-admin-architecture`
- backend -> `nest-api-architecture`
- schema e relacoes -> `database-design`
- login, sessao, papel e acesso -> `auth-rbac`
- pedido, status, estoque e capacidade -> `order-operations`
- checkout, Stripe, webhook ou cobranca -> `payments-integration`
- push, SMS ou atualizacao de pedido -> `realtime-notifications`
- aparencia -> `frontend-design`
- qualidade -> `testing-strategy`
- logs e suporte -> `observability-support`
- deploy, ambientes ou CI/CD -> `deployment-infra`

Regra:

- escolha uma skill principal
- use no maximo duas se houver dependencia real
- justifique a escolha no Task Contract
